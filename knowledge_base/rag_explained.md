# RAG（检索增强生成）详解

## 什么是 RAG
RAG（Retrieval Augmented Generation）是一种让 LLM 能够访问外部知识的技术。核心思路：先检索相关文档，再让 LLM 基于检索到的内容生成回答。

## 为什么需要 RAG
LLM 存在以下局限：
1. **知识截止日期**：训练数据有时间限制，不知道最新信息
2. **幻觉问题**：LLM 可能编造不存在的事实
3. **领域知识不足**：对特定领域（如公司内部文档）缺乏了解
4. **上下文限制**：不能一次性读取所有文档

RAG 通过检索相关文档片段，将其作为上下文提供给 LLM，解决了这些问题。

## RAG 的工作流程
1. **文档加载**（Document Loading）：读取 PDF、Markdown、网页等格式的文档
2. **文档切分**（Chunking）：将长文档切成 200-1000 字的小块
3. **向量化**（Embedding）：将文本块转换为向量（一组数字）
4. **存储**（Storage）：将向量存入向量数据库（如 ChromaDB、FAISS）
5. **检索**（Retrieval）：用户提问时，将问题也向量化，找到最相似的文本块
6. **生成**（Generation）：将检索到的文本块 + 用户问题一起发给 LLM 生成回答

## 向量化（Embedding）是什么
Embedding 是将文本转换为数字向量的过程：
- "猫" → [0.2, 0.8, -0.1, ...]
- "狗" → [0.3, 0.7, -0.2, ...]
- "汽车" → [-0.5, 0.1, 0.9, ...]

语义相近的词，向量也相近。通过计算向量之间的距离（余弦相似度），可以找到语义最相关的文档片段。

## 常用工具
- **LlamaIndex**：最专业的 RAG 框架，开箱即用
- **ChromaDB**：轻量级向量数据库，适合本地使用
- **FAISS**：Facebook 的向量搜索库，性能强
- **HuggingFace Embeddings**：免费的中文 Embedding 模型
